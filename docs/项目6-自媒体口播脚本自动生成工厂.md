# 项目6：自媒体口播脚本自动生成工厂

**时长**：7天
**难度**：高阶
**简历价值**：辅助条目，"多Agent编排+MCP"

---

## 业务场景

短视频MCN机构每天要产出大量口播脚本：选题→搜集素材→写稿→审核→修改。纯人工一天产3-5篇，质量波动大。需要一个多Agent流水线来提效。

## 用户画像

短视频MCN内容团队（5-10人），管理20+账号。痛点不是不会写，是**产量跟不上**和**风格一致性差**——A编导写搞笑号，B编导写情感号，但A休假时B代写，风格就变了。

## 为什么选这个场景

多Agent协作最适合的场景就是**有天然分工的创作流水线**。策划→写作→审核→修改，每个环节是独立Agent，有明确输入输出。不像"多Agent聊天演示"那样牵强，这个分工是真实工作流。

---

## 功能

- 输入：选题（如"打工人如何保持精力"）+ 账号风格 + 时长
- 策划Agent：分析选题 → 查热点 → 出大纲+金句
- 写作Agent：按大纲+风格写口播稿
- 审核Agent：检查事实/风格一致/平台合规/查重
- 修改Agent：接收审核意见 → 逐条修改 → 输出终稿
- 编排器：控制串行流程，某Agent失败→重试或降级
- 输出：完整口播脚本 + 拍摄建议

---

## 五层实现

### 层1：跑通
- Agent消息schema设计（Pydantic）：

```python
class Outline(BaseModel):
    title: str
    sections: list[Section]
    key_phrases: list[str]

class Script(BaseModel):
    content: str
    duration_estimate: int

class ReviewResult(BaseModel):
    passed: bool
    issues: list[Issue]
    score: float
```

- 四个Agent独立prompt设计：
  - 策划Agent：角色+搜索工具+输出大纲格式
  - 写作Agent：角色+风格参数+口播节奏要求
  - 审核Agent：角色+合规规则（广告法极限词等）+检查清单
  - 修改Agent：角色+逐条对照修改指令
- 编排器状态机：策划→写作→审核→(修改→审核)×N→输出
  - 最多3轮修改，超限→标记需人工介入
- MCP搜索工具接入（获取热点/事实核查）
- Agent间通过Pydantic schema传递结构化消息

### 层2：评估
- 30个选题，覆盖3种账号风格（搞笑/知识/情感）
- 对比实验：单Agent（一个模型直接写全稿）vs 多Agent流水线
- 人工评估维度：
  - 信息量（有没有实质性内容）
  - 口语化（能不能读出嘴）
  - 合规性（有没有极限词/违规内容）
  - 可用率（能不能直接拿去拍）
- 记录平均修改轮次

### 层3：护栏
- Agent间消息格式Pydantic校验
- 无限循环检测：修改→审核循环最大3轮
- 输出合规检查：
  - 广告法极限词检测（"最好""第一""国家级"等）
  - 医疗健康类内容敏感审核
  - 政治敏感内容过滤
- 审核Agent评分过低（<60分）时自动标记人工

### 层4：观测
- 每个Agent耗时/Tokens/重试次数
- 整条流水线的总耗时
- 审核通过率（一次通过 vs 需要修改）
- 修改轮次分布

### 层5：部署
- Docker Compose：四个Agent独立服务+编排器服务
- FastAPI统一入口
- README含架构图+部署说明
